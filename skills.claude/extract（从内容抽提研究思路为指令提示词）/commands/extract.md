# /extract - 研究方法论反向提取器

**描述**: 从已有的深度研究文档中抽象出可复用的研究框架模板，实现"研究思路模板化、未来场景复用化"

**使用方法**:
```
/extract [研究文档路径]
```

**示例**:
```
/extract 深度研究/Unifuncs.md
/extract 深度研究/MiroThinker.md
/extract 2026年核心资产投资研究报告.md
```

---

## 工作原理

当你执行 `/extract` 命令时，系统会：

1. **读取并分析**指定的研究文档
2. **识别并提取**六个核心维度的研究方法论框架：
   - 宏观分析框架（历史对标、时代特征、关键时间点）
   - 资产分类框架（分类逻辑、驱动因素、配置角色）
   - 驱动逻辑验证框架（必要条件、转折信号）
   - 三重验证体系（宏观+产业+财务）
   - 风险预警框架（双向风险、动态调整）
   - 专家观点整合框架（观点交叉验证）

3. **生成并保存**可复用的研究方法论模板到 `research_frameworks/` 目录

---

## 核心价值

### 不是提取结论，而是提取方法

❌ **错误理解**：
- "提取2026年黄金的投资逻辑"
- "总结三硬资产的投资策略"

✅ **正确理解**：
- "提取如何进行宏观对标的分析方法"
- "总结如何构建三重验证体系的研究框架"

### 实现跨场景复用

**原场景**: 2026年"三硬资产"投资策略研究

**复用场景**:
- 2027年"三硬资产"更新研究 → 直接套用框架，更新数据
- 2026年"软资产"研究 → 复用六维度结构，调整分类逻辑
- 其他宏观主题研究 → 复用分析范式，定制化内容

## 使用示例

### 示例1: 提取单个报告的方法论

```bash
/extract 深度研究/Unifuncs.md
```

系统会：
1. 分析Unifuncs.md的研究逻辑
2. 提取"三硬资产"分类方法论
3. 提取"必要逻辑链"验证方法
4. 生成8个框架模板文件

---

### 示例2: 对比提取多个报告

```bash
/extract 深度研究/Unifuncs.md,深度研究/MiroThinker.md
```

系统会：
1. 分别分析两个报告
2. 对比它们的研究方法论
3. 提取共性框架和差异点
4. 生成融合版方法论模板

---

### 示例3: 提取特定维度

```bash
/extract 深度研究/Unifuncs.md --dimensions=macro,verification
```

系统会：
1. 只提取宏观分析框架（macro）
2. 只提取三重验证体系（verification）
3. 其他维度跳过

## 注意事项

1. **文档要求**: 输入的研究文档需要有一定的分析深度和结构化程度
2. **提取范围**: 默认提取全部六个维度，可通过参数指定
3. **输出位置**: 所有框架保存在 `research_frameworks/` 目录
4. **版本管理**: 每次提取都会创建带时间戳的新目录，旧版本保留
5. **人工审核**: 提取的框架需要人工审核和调整后使用

## 常见问题

**Q: 提取的框架可以直接使用吗？**
A: 建议先人工审核和调整，确保符合您的具体需求。

**Q: 可以提取非投资类的研究文档吗？**
A: 可以！本技能适用于任何有逻辑深度的分析报告。

**Q: 如何判断提取质量？**
A: 查看生成的 `00_研究方法论总览.md`，检查是否准确反映了原报告的研究思路。

**Q: 框架可以分享给团队吗？**
A: 可以！整个 `research_frameworks/` 目录都可以分享和版本控制。
